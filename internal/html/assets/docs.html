Learn how to create encrypted recovery bundles and recover your secrets

Prefer the command line? See the CLI guide.

Overview

ReMemory helps you protect important files (passwords, crypto keys, documents) by:

  1. Encrypting your files with strong cryptography (age encryption)
  2. Splitting the decryption key among trusted friends using Shamir's Secret Sharing
  3. Creating self-contained bundles that friends can use to recover your secrets

The important part: recovery works entirely offline in a browserâ€”no servers, no internet, no need for this website to exist when recovery happens.

Tip: No single friend can access your data alone. They must work together, combining their shares to reach the threshold you set (e.g., 3 of 5 friends).

Creating Bundles

Creating bundles is a 3-step process. Everything happens in your browserâ€”your files never leave your device.

Step 1: Add Friends

Start by adding the friends who will hold pieces of your recovery key. For each friend, provide their name and optionally contact information.

Then choose your thresholdâ€”how many friends must cooperate to recover your files.

Choosing the right threshold:
  • 3 friends, threshold 2: Minimum viable setup
  • 5 friends, threshold 3: Good balance of security and availability
  • 7 friends, threshold 4-5: Higher security, requires more coordination
Set threshold high enough that casual collusion is unlikely, but low enough that recovery is possible if 1-2 friends are unavailable.

Step 2: Add Files

Drag and drop files or a folder containing the secrets you want to protect.

Good candidates for ReMemory:

  • Password manager recovery codes
  • Cryptocurrency seeds/keys
  • Important account credentials
  • Instructions for loved ones
  • Legal document locations
  • Safe combinations
Note: Avoid files that change frequently. ReMemory is designed for static secrets that you set and forget.

Step 3: Generate Bundles

Click "Generate Bundles" to encrypt your files and create personalized bundles for each friend.

Each bundle is approximately 5 MB because it includes the complete recovery tool. This ensures friends can recover your files even if this website no longer exists.

Distributing to Friends

Send each friend their specific bundle using any method you trust:

  • Email: Attach the ZIP file
  • Cloud storage: Share via Dropbox, Google Drive, etc.
  • USB drive: Physical handoff
  • Encrypted messaging: Signal, WhatsApp, etc.
Tell your friends:
  1. Keep the bundle somewhere safe (cloud backup, USB drive, etc.)
  2. They cannot use it aloneâ€”they'll need to coordinate with others
  3. A single share reveals nothing, but they should still keep it private
  4. Consider printing the README.pdf and keeping it in a safeâ€”paper survives digital disasters. The MANIFEST.age file can be stored separately (cloud, email, USB) since it's just encrypted ciphertext.

Recovering Files

What Friends Receive

Each bundle contains everything needed for recovery:

README.txt Instructions + unique share + contact list
README.pdf Same content, formatted for printing. Also includes a QR code that can be scanned to import the share directly.
MANIFEST.age Your encrypted secrets
recover.html Browser-based recovery tool (~1.8 MB)

Each bundle is personalized: the recover.html has the friend's share pre-loaded and shows a contact list of other friends who hold shares. When the encrypted manifest is 5 MB or less, it is also embedded in recover.html, so friends only need to collect shares from others to complete recovery.

Path A: I Have the Bundle ZIP

This is the easiest path. If you have the bundle ZIP file (or the individual files from it), here's what to do:

1

Extract the ZIP and open recover.html

Open the recovery tool in any modern browser. Your share is automatically pre-loaded.

2

Load the encrypted manifest

For small manifests (â‰¤ 5 MB), this step is automaticâ€”the manifest is already embedded in recover.html. Otherwise, drag and drop MANIFEST.age from the bundle onto the manifest area.

3

Coordinate with other friends

The tool shows a contact list with other friends' names and contact information. Reach out and ask them to send their README.txt file or share text.

4

Add shares from other friends

For each friend's share, you can: drag and drop their README.txt file, paste share text directly, or scan a QR code from their printed PDF. A checkmark appears next to each friend's name as their share is added.

5

Recovery happens automatically

Once the threshold is met (e.g., 3 of 5 shares), decryption starts immediatelyâ€”no button to click!

Pro tip: If a friend sends you their entire .zip bundle file, you can drag it directly onto the pageâ€”the manifest and their share are both imported automatically.

Path B: I Have a Printed PDF

Some people prefer to keep just the PDF printout in a safe or filing cabinet and store the MANIFEST.age file separately (e.g. cloud storage, email, USB drive). Since the manifest is encrypted, it's safe to keep copies anywhereâ€”nobody can open it without combining enough shares.

If you're starting from a printed PDF:

1

Open the recovery tool

Scan the QR code on the PDF with your phone cameraâ€”it opens the recovery website with your share pre-filled. Alternatively, visit the URL printed on the PDF and type the short code (RM1:...) shown below the QR code into the paste area.

2

Load the encrypted manifest

You need the MANIFEST.age fileâ€”drag it onto the page or click to browse. If you don't have it, ask one of the other friends to send it from their bundle. Every bundle contains the same copy.

3

Collect shares from other friends

Contact other friends and ask them to send their shares. They can send their README.txt, share text, or you can scan the QR code from their printed PDF.

4

Recovery happens automatically

Once the threshold is met, decryption starts immediately.

Key points about recovery:
  • Works completely offlineâ€”no internet required
  • No data leaves the browser
  • Works on Chrome, Firefox, Safari, Edge
  • Friends can be in different locations; they just share their README.txt files

Best Practices

Choosing Friends

  • Longevity: Pick people likely to be reachable in 5-10+ years
  • Geographic diversity: Don't put all friends in the same disaster zone
  • Technical ability: Mix is fine; the tool is designed for everyone
  • Relationships: Consider if they'll cooperate with each other
  • Trust: While a single share reveals nothing, you're trusting them with responsibility

Security Considerations

  • Don't keep all bundles togetherâ€”that defeats the purpose of splitting
  • Consider printing README.pdfâ€”paper backups survive digital disasters
  • Save project.yml if you want to regenerate bundles later

Rotation

Consider creating new bundles every 2-3 years:

  • Friends' contact info changes
  • You may want to update secrets
  • Relationships change
  • New cryptographic best practices emerge

Advanced: Anonymous Mode

For situations where you don't want shareholders to know each other's identities, ReMemory offers an anonymous mode. In this mode:

  • Friends are labeled generically as "Share 1", "Share 2", etc.
  • No contact information is collected or stored
  • READMEs skip the "Other Share Holders" section
  • Bundle filenames use numbers instead of names

When to Use Anonymous Mode

Anonymous mode is useful when:

  • You want to distribute shares to people who shouldn't know each other
  • You're testing the system quickly without entering contact details
  • You have a separate out-of-band method for coordinating recovery
  • Privacy is a higher priority than ease of coordination

How to Enable

In the bundle creator, look for the Anonymous toggle in the top-right corner of the Friends section. When enabled:

  • The friend list disappears
  • A "Number of shares" input appears instead
  • Set how many shares you want and the threshold
  • Generated bundles will be named bundle-share-1.zip, bundle-share-2.zip, etc.

Recovery in Anonymous Mode

Recovery works the same way, but the contact list section won't appear in recover.html. Share holders will see generic labels like "Share 1" instead of names.

Important: Since there's no built-in contact list in anonymous mode, make sure share holders know how to reach each other (or you) when recovery is needed. You'll need to coordinate through other means.

Advanced: Multilingual Bundles

Each friend can receive their bundle (README.txt, README.pdf, and recover.html) in their preferred language. ReMemory supports 5 languages: English, Spanish, German, French, and Slovenian.

How It Works

  • In the web UI, each friend entry has a Bundle language dropdown
  • If set to "Default", the friend's bundle uses the current UI language
  • Override per friend to generate bundles in different languages
  • The recover.html tool will open in the friend's selected language by default
  • Friends can always switch languages in recover.html regardless of the default

CLI Usage

Set the project-level default language:

rememory init my-recovery --language es

Override per friend using the extended --friend format:

rememory init my-recovery --language es \
  --friend "Alice,alice@example.com,en" \
  --friend "Roberto,roberto@example.com,es"

Or set it directly in project.yml:

language: es
friends:
  - name: Alice
    contact: alice@example.com
    language: en
  - name: Roberto
    contact: roberto@example.com
